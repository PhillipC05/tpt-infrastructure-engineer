### How to run locally:

#### 0. __Prerequisites__

- Python 3.12+
- Node.js 20+
- PostgreSQL 16 with PostGIS 3.4

Copy the environment template and adjust as needed:
```bash
cp backend/.env.example .env
```

#### 1. __Install root dependencies__

```bash
npm install
```

#### 2. __Start Backend API__

```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --reload
```

✅ Backend will run at `http://localhost:8000` ✅ API Documentation: `http://localhost:8000/docs`

#### 3. __Start Frontend__

```bash
cd frontend
npm install
npm run dev
```

✅ Frontend will run at `http://localhost:5173`

---

### Database Setup

PostgreSQL must be running with PostGIS extension. Create the database:

```cmd
psql -U postgres -c "CREATE DATABASE tpt_infrastructure;"
```

Tables are managed via Alembic migrations. To apply:
```bash
cd backend
alembic upgrade head
```

For development, the application will auto-create tables on first run.










Postgres

### 1. First, start PostgreSQL Service on Windows:

Run this command in Administrator Command Prompt:

```cmd
net start postgresql-x64-16
```

*(Note: Version number may differ - if that fails, check services with `services.msc` and find the correct PostgreSQL service name)*

### 2. Verify PostgreSQL is running:

Check port 5432 is listening:

```cmd
netstat -ano | findstr :5432
```

### 3. Create the database:

Once PostgreSQL is running, create the database first:

```cmd
psql -U postgres -c "CREATE DATABASE tpt_infrastructure;"
```

*(Default password will be `postgres` as configured in your connection string)*

### 4. Apply database migrations:

```bash
cd backend
alembic upgrade head
```

Or let the application auto-create tables on first run during development.
